WHAT IS THIS?

The Wonder Cabinet is a physical arcade machine built around an unlikely constraint: a single 64×64 RGB LED matrix — just 4,096 pixels. Each pixel is a real, glowing LED. No anti-aliasing, no transparency, no GPU shaders. Every game, simulation, and visualization was designed from scratch to be beautiful at this resolution.

Inside are dozens of playable games, hundreds of visual experiences, a gallery of classical paintings, and a set of AI-controlled game demos, all running on a Raspberry Pi. The name comes from the Renaissance tradition of Wunderkammern — cabinets of curiosities that gathered natural specimens, scientific instruments, and works of art into a single room of wonder.

The constraint is the point. On a 4K display, a protein molecule is just another render. On 64×64 glowing LEDs, it becomes something you stare at. The resolution forces every visual to be distilled to its essence — what are the minimum pixels needed to recognize a double helix? To see a flock emerge? To feel fire rising?
How to read this guide. The sections below mirror the cabinet's own menu, category for category, in the same order you'll scroll through them on the machine. Every entry lists what it is, where the idea came from, and the controls that drive it. This page is generated directly from the cabinet's software — so it always matches what's actually installed.

USING THE CABINET

The whole machine is driven by one joystick and two buttons. Both buttons do the same thing in almost every game — either one is “the button.” A handful of two-handed games (pinball flippers, Portal's two colors) are the only exceptions, and those are noted in each entry's controls.

GETTING AROUND THE MENU

  1. Switch category with LEFT / RIGHT. The categories are the same ones listed in this guide — games first, then art, science, music, and the rest.
  2. Pick an item within a category with UP / DOWN.
  3. Launch it with either button.
  4. Come back by holding both buttons together for about two seconds. This works from inside any game or visual and drops you back at the menu — the on-screen hint reads HOLD:EXIT.
Leave it alone and it comes alive. After about a minute of no input, the cabinet drifts into an idle screensaver — it cycles through a shuffled, weighted mix of the visuals, cross-fading between them. Touch any control and it snaps back to the menu. You can tune all of this: which visuals appear and how often, how long before idle starts, and which transition effects are used.

SETTINGS & MAINTENANCE

The cabinet's own settings live in the UTILITY category at the end of the menu — brightness and gamma, the idle-screensaver mixer and timers, accessibility options, WiFi setup, a pixel-test diagnostic, software updates, and a safe shutdown. Every one of them is documented in the UTILITY section below, just like the games and visuals.

Powering down. The cabinet runs on a Raspberry Pi, so don't just yank the cord — open UTILITY › SHUTDOWN to power it down cleanly first, then switch off the power. To update to the latest software over WiFi, use UTILITY › UPDATE, which can also roll back if an update misbehaves.

HOW IT WORKS

64x64 LED Array + Raspberry Pi + Joystick + 2 Buttons + Power Supply

DISPLAY 64×64 RGB LED matrix
HUB75 interface
4,096 individually addressable LEDs
COMPUTER Raspberry Pi 3
1.2 GHz ARM Cortex-A53
1 GB RAM
SOFTWARE Python 3
~40,000 lines of code
Every effect built from scratch
INPUT Joystick + 2 buttons
Wired to GPIO pins
Hold both to exit
No GPU. No shaders. Every pixel is computed in Python on a 1.2 GHz ARM chip. Fluid dynamics, ray tracing, classic arcade games, reaction-diffusion, N-body gravity — all running at 30fps on tiny, concise physical hardware.
